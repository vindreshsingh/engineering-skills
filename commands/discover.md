Follow the `product-discovery` skill (`skills/product-discovery/SKILL.md`). Run the PM persona trio
through the Define and Plan phases: frame the real problem (business-analyst / `idea-shaping`), write
the PRD with scope and acceptance criteria (product-manager / `product-brief`), define the success
metric (product-analyst / `observability`), decide the RFC handoff, and produce an ordered task
breakdown (`work-planning`). Stop for sign-off at the Define and Plan gates. Output linked artifacts,
not code.
