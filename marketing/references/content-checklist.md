# Content Quality Checklist

Quick reference to run alongside [[content-marketing]]. Use before publishing any long-form piece.

## Structure

- [ ] Hook in first 100 words — specific pain or surprising stat
- [ ] Problem → insight → solution → proof → CTA flow
- [ ] H2/H3 headings; scannable bullets and short paragraphs
- [ ] One primary CTA (mid + end max)

## Developer audience

- [ ] 80% value, 20% product — useful even without sign-up
- [ ] Concrete examples, steps, or commands — not vague advice
- [ ] `[SCREENSHOT]` / `[GIF]` placeholders noted where needed
- [ ] Honest about what the product does *not* do yet

## SEO (if applicable)

- [ ] Title includes primary keyword + benefit
- [ ] Meta description 150–160 chars with CTA verb
- [ ] Primary keyword in H1 and first 200 words
- [ ] 2–3 internal links with descriptive anchor text

## Repurposing

- [ ] 3 pull quotes / thread bullets drafted for social
- [ ] Repurpose table filled (X, LinkedIn, Reddit, newsletter)

## Quality gate

- [ ] Would I share this if I didn't work on the product?
- [ ] Every claim true or marked `[VERIFY]`
- [ ] No press-release fluff or empty superlatives
