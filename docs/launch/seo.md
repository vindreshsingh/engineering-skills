# SEO & discovery

_Drafted as the `seo-strategist` persona using the `seo-growth` skill. For an open-source repo, "SEO"
is mostly GitHub discovery + inbound links._

## Repo metadata (done — live)

- **Description:** keyword-rich, leads with the differentiator and the supported tools.
- **Topics:** `ai`, `ai-agents`, `agent-skills`, `claude-code`, `llm`, `coding-agent`, `sdlc`,
  `skills`, `cursor`, `codex`, `gemini`, `software-development`, `tdd`, `developer-tools`,
  `prompt-engineering`.

## Target search terms (what people actually type)

| Intent | Terms |
|--------|-------|
| Discovery | "claude code skills", "agent skills", "ai coding agent skills" |
| Comparison | "superpowers alternative", "claude code plugins" |
| Task | "tdd skill for ai agent", "ai agent code review skill" |
| Tool | "cursor rules engineering", "codex AGENTS.md skills" |

Mirror these phrases naturally in the README H1/intro, the announcement post title, and section
headings — that's what gets indexed.

## Inbound links (highest-leverage off-repo work)

- Submit a PR to **awesome-claude-code**, **awesome-ai-agents**, **awesome-cursor** style lists.
- Get into plugin **marketplace listings** (each is an indexed inbound link + install surface).
- The announcement post + Show HN + threads each link back with anchor text using the target terms.

## On-repo SEO

- README first paragraph should contain the primary phrases ("agent skills", "Claude Code", "Cursor",
  "Codex") in natural prose — it's the snippet search shows.
- Keep `SKILLS.md` (the catalog) — it's a keyword-dense, crawlable index of every skill.
- A short, specific README beats a long vague one for ranking and for humans.

## Measurement

- GitHub **traffic insights** (referrers + popular content) weekly.
- UTM tags on every shared link → see which channel actually drives visits/installs.
