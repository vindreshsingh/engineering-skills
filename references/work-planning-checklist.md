# Work Planning Checklist

Quick reference alongside [[work-planning]].

## Inputs

- [ ] Spec or PRD with acceptance criteria
- [ ] Known constraints (deadline, team, dependencies)

## Breakdown

- [ ] Tasks ordered by dependency
- [ ] Each task verifiable independently
- [ ] Vertical slices preferred over horizontal layers
- [ ] Spikes called out for unknowns

## Estimates

- [ ] Rough estimate per task or slice
- [ ] Assumptions written down
- [ ] Risks flagged (external deps, new tech)

## Handoff

- [ ] Plan readable by another engineer
- [ ] First task clear to start immediately
- [ ] Out of scope listed explicitly
