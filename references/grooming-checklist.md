# Grooming Checklist

Quick reference alongside [[product-grooming]].

## Session setup

- [ ] Sprint target identified (usually next sprint)
- [ ] Timebox set (60–90 min)
- [ ] Backlog ordered by priority
- [ ] Carry-over "Not ready" items reviewed first

## Per story

- [ ] User story rewritten (As a … I want … so that …)
- [ ] Acceptance criteria testable (Given/When/Then or checklist)
- [ ] In scope / out of scope explicit
- [ ] Split if > one sprint slice
- [ ] Architecture feasibility checked
- [ ] Security flags noted (PII, authz)
- [ ] Rough estimate with assumptions
- [ ] Dependencies listed (schema, API, UI order)
- [ ] Open questions have owner + due date

## Definition of Ready

- [ ] Ready or Not ready verdict recorded
- [ ] Not ready: missing items listed
- [ ] QA voice confirms AC testable
- [ ] Eng manager confirms capacity fit

## Session close

- [ ] Prioritized list for sprint planning
- [ ] Session notes shared with scrum-master
- [ ] No blocking questions left unowned
